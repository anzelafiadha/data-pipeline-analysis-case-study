# Data pipeline analysis (case study)
This project presents a simplified case study of an ETL (Extract, Transform, Load) process using a fully generated dataset.

The project then applies a simple ETL workflow, starting from data generation (extract), followed by data cleaning and transformation (transform), and preparing structured data for analysis (load).

Finally, a set of analytical queries is used to explore the dataset and extract insights, such as student distribution across schools and identification of data quality issues.
